Які локальні порти відкриті на вашому комп'ютері (LISTENING)?
-

Порт 135, 445, 5040, і т.д


Що означають стани: ESTABLISHED, TIME_WAIT, CLOSE_WAIT, LISTEN?
-

ESTABLISHED - Оба адреса зробили handshake та знають про друг-друга


TIME_WAIT - Коли адреса кидає FIN, вона входить в цей стан для того щоб закритись. Це робиться 
для того щоб відкинути якісь пакети які затримались, або не дійшли


CLOSE_WAIT - Коли система вже знає що з'єднання закрився.


LISTEN - Адреси чекають на handshake

Яка різниця між портами 0-1023, 1024-49151 та 49152-65535?
-

0-1023 - це системні порти

1024-49151 - це зареєстровані порти

49152-65535 - це порти які призначуються ОС новим з'єднанням

Чому HTTP використовує порт 80, а HTTPS - 443?
-

Бо це не однакові протоколи

HTTP - це протокол для веб-серверів

HTTPS - теж, але більш захищений протокол. він був зареєстрований пізніше

Що таке ephemeral ports (тимчасові порти)?
-

Тимчасові порти - це порти, які ОС автоматично призначає для мережевих з'єднань.

Вони знаходяться в діапазоні 49152-65535

Чи можна запустити два сервери на одному порту одночасно?
-

Ні, не можна. Два сервери не можуть одночасно використовувати один і той самий порт, так як він може бути зайнятий

Завдання 3: Експерименти з TCP з'єднаннями
-

Що відбувається під час встановлення TCP з'єднання (3-way handshake)?
-

Відбувається обмін трьома службовими пакетами між клієнтом і сервером, щоб обидві сторони домовились про з'єднання.

Як сервер розуміє, що клієнт хоче з'єднатися?
-

Після того як клієнт кидає SYN-пакет, що буквально означає: "Я хочу під'єднатись"

Що станеться, якщо порт закритий або фільтрується firewall?
-

Якщо клієнт кидає SYN і відмовляє, сервер кидає RST.

Якщо firewall блокує, то з'єднання відмовиться через деякий час.
